# Brainjob

Brainjob is a JSON-only job-search information system. Employer content stays immutable in `job.json`; your application workflow, notes, contacts, and documents live in separate files you edit by hand.

No database, YAML, or frontend build step is required.

## Layout

```
brainjob/
├── data/
│   ├── jobs/<job-id>/          # authoritative records (human-editable JSON)
│   └── templates/job/          # templates used by `brainjob add`
├── schemas/                    # formal JSON Schema contracts (Draft 2020-12)
├── tracking/                   # generated by `brainjob sync`
│   ├── index.json
│   ├── dashboard.html
│   └── assets/
├── src/brainjob/               # Python CLI
└── tests/
```

Each job directory contains:

| File | Purpose |
|------|---------|
| `job.json` | Vacancy record; `description_original.content` is never modified by tooling |
| `application.json` | Status, timeline, interviews, assessments, offer |
| `contacts.json` | People linked to the vacancy |
| `notes.json` | Personal notes (`author`: `user` or `ai`) |
| `documents.json` | CV/cover letter metadata and submission state |

## Install

```bash
cd brainjob
python -m pip install -e ".[dev]"
```

## Commands

Run all commands from the `brainjob/` workspace root (or pass `--root` / set `BRAINJOB_ROOT`).

### `brainjob add`

Create a new job from templates:

```bash
brainjob add \
  --title "Policy Officer" \
  --company "Example Company" \
  --description "Paste the full original posting here." \
  --url "https://example.com/jobs/policy-officer" \
  --location "Brussels, Belgium" \
  --city Brussels \
  --country Belgium \
  --deadline 2026-08-15 \
  --tags policy eu-affairs
```

`brainjob add` stamps `description_original.sha256` at capture time. After that, tooling verifies but never rewrites the original description text.

### `brainjob validate`

Validate every active job directory under `data/jobs/`:

```bash
brainjob validate
```

Checks JSON shape, cross-file `job_id` consistency, and SHA-256 integrity of `description_original.content`.

### `brainjob sync`

Regenerate `tracking/index.json` and `tracking/dashboard.html`:

```bash
brainjob sync
```

Sync refuses to run when validation fails.

### `brainjob sync --check`

Exit 0 when `tracking/index.json` matches current job data; exit 1 when missing or stale:

```bash
brainjob sync --check
```

### `brainjob watch`

Poll `data/jobs/` for JSON changes and run sync automatically:

```bash
brainjob watch
```

### `brainjob archive <job-id>`

Mark the application archived, append a timeline event, and move the job to `data/jobs/_archive/<job-id>/`:

```bash
brainjob archive example-company-policy-officer
```

Archived jobs are excluded from the generated index and dashboard.

### `brainjob status`

Print a concise pipeline summary (job counts by status, overdue next actions, upcoming deadlines) without writing tracking files:

```bash
brainjob status
```

Exits 1 when validation fails; overdue items are informational and do not fail the command.

## Dashboard

| Access | Link |
|--------|------|
| Live | [https://brickwalllegend2000.github.io/jobbrainmine/](https://brickwalllegend2000.github.io/jobbrainmine/) |
| Local | `http://localhost:8000/dashboard.html` (after the commands below) |

After `brainjob sync`, serve the tracking folder locally:

```bash
python -m http.server 8000 --directory tracking
```

Then open `http://localhost:8000/dashboard.html` on the same machine.

On `main`, CI regenerates `tracking/` and publishes the dashboard to GitHub Pages at [https://brickwalllegend2000.github.io/jobbrainmine/](https://brickwalllegend2000.github.io/jobbrainmine/).

The dashboard includes:

- Pipeline stats and status filters
- Overdue next actions and upcoming deadlines
- Full original job description view
- Application timeline, interviews, and assessments
- Contacts, notes, and document submission tracking
- Compensation, location, source link, and capture timestamps
- Original-content integrity indicator (SHA-256 verification)

Data is embedded at sync time from authoritative JSON under `data/jobs/`.

## Immutability rules

1. `description_original.content` in `job.json` is treated as read-only employer content.
2. Brainjob verifies content with `description_original.sha256` and reports mismatches.
3. Personal and AI-generated text belongs in `notes.json` (AI notes use `"author": "ai"`).
4. `tracking/index.json` is generated; never treat it as the source of truth.

## Development

```bash
python -m pip install -e ".[dev]"
python -m pytest
ruff check src tests
brainjob validate
brainjob sync
```

### Pre-commit hooks

From the repository root (after `pip install -e ".[dev]"` in `brainjob/` so `brainjob` is on `PATH`):

```bash
pre-commit install
```

Hooks run ruff on Python under `brainjob/`, plus `brainjob validate` and `brainjob sync --check` when job or tracking files change.
